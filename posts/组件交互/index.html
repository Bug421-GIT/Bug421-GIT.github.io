父传子 props

​ 父组件传递给子组件的数据，子组件通过props接受后，不能直接更改该值。这是因为：

  1. 所有的 prop 都使得其父子 prop 之间形成了一个单向下行绑定：父级 prop 的更新会向下流动到子组件中
  2. 但是反过来则不行。这样会防止从子组件意外变更父级组件的状态，从而导致你的应用的数据流向难以理解

子传父 自定义事件

使用场景：子组件给父组件传值 — 父组件给子组件绑定自定义事件，子组件触发事件，回调函数在父组件中

  1. 绑定事件 $emit("eventName",parmas)

  2. 解绑事件 $off("eventName")

  3. 监听事件 $on("eventName",parmas)

$emit

给谁绑定事件，找谁触发事件，找谁解绑事件

$on

​ 只触发一次 $once 或者 eventName.once

  1. 自定义事件的回调函数，this指向被绑定、被触发的组件
  2. 在当前组件调用methods定义的函数时，已当前组件函数的this为主
  3. 生命周期钩子以普通函数方式定义，this一定取当前组件

    .$on("eventName",this.getChildrenName) // 在当前组件调用methods定义的函数，以当前组件this为主
    
    this.$refs.children // 这里的this指向children组件
    .$on("eventName",function(name, ...params){ // 新定义的函数，指在children只能定义一个函数，参数取值都在children中
		console.log(this);
    })
    
    this.$refs.children // 这里的this指向children组件, 外层的this指向father
    .$on("eventName",(name,...params)=>{ // 箭头函数往外层找this
        this.sonValueName = name;
    })
}

【总结】使用$ref为组件绑定自定义事件时，必须关注this的指向 — 回调要么配置在methods中，要么使用箭头函数

兄弟间传值 全局事件总线（GlobalEventBus）

​ 适用于任意组件间的通信

①所有组件都能看到它 的有三种：

  1. window — 不应该修改
  2. VueComponent — 仅一个构造函数，需要调用Vue.extend生成
  3. vc — 每次生成都是一个新的实例，互不关联
  4. Vue — 将x放在 Vue的原型对象上 Vue.prototype

VueComponent.prototype._proto_ === Vue.prototype 组件实例对象Vc可以访问到Vue原型上的属性、方法

②它可以调用所有组件的事件：

  1. 基础数据类型，不含括$on $off $emit

  2. VueComponent — vue实例vm无法调用

    1. const Demo = Vue.extend({config}) — Demo 是Vue.extend生成一个VueComponent 构造函数
    2. const demo = new Demo — demo 是通过new 创建的vc实例对象
    3. 相当于在页面中引入一个组件<vm />
    4. Vue.prototype.x = demo — Vue的原型对象上配置一个vc对象，拥有调用组件事件的函数

  3. vue实例

    1. Vue 创建前绑定，没有vue实例对象可以赋值；Vue 创建完成后绑定，页面已经完成挂载，子组件调用不到

    2. Vue beforeCreate()时绑定，且指向自身（无需新增变量）

      初始化时，Vue数据检查、数据代理未开始，页面未挂载
